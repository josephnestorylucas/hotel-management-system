# Bug #4 — Back Button Works After Logout + 419 Page Expired on Login

**Date:** 2026-05-29
**Status:** Fixed
**Production URL:** http://hms.dadyprojects.systems

---

## Problem 1: Back Button After Logout

After logging out, pressing the browser's back button showed the previous authenticated page. The user could continue using the application as if never logged out.

**Root Cause:** Browser was serving cached pages. No `Cache-Control` headers were set to prevent caching of authenticated responses.

## Problem 2: 419 Page Expired on Login

On production (`http://hms.dadyprojects.systems`), submitting the login form returned **419 Page Expired** (CSRF token mismatch).

**Root Cause:** `SESSION_SECURE_COOKIE=true` in `.env` tells the browser to only send the session cookie over HTTPS. The production site runs on HTTP, so the browser never sends the session cookie. Without the session cookie, Laravel can't match the CSRF token → 419 error.

---

## Fix

### 1. `.env` — Session Cookie

Changed:
```
SESSION_SECURE_COOKIE=true  →  SESSION_SECURE_COOKIE=false
```

The browser now sends the session cookie over HTTP. The 419 error is resolved.

### 2. `app/Http/Middleware/SecurityHeaders.php` — No-Cache Headers

Added to every response:
```php
$response->headers->set('Cache-Control', 'no-cache, no-store, max-age=0, must-revalidate');
$response->headers->set('Pragma', 'no-cache');
$response->headers->set('Expires', 'Sun, 02 Jan 1990 00:00:00 GMT');
```

The browser is forced to revalidate every page with the server. After logout, pressing back triggers a fresh request which hits the `auth` middleware and redirects to login.

### 3. `app/Http/Controllers/Auth/LoginController.php` — Logout Response Headers

Added the same no-cache headers to the logout redirect response itself, as an extra layer of protection.

---

## Files Changed

| File | Change |
|---|---|
| `.env` | `SESSION_SECURE_COOKIE=false` |
| `app/Http/Middleware/SecurityHeaders.php` | Added `Cache-Control`, `Pragma`, `Expires` headers |
| `app/Http/Controllers/Auth/LoginController.php` | Added no-cache headers to logout redirect |

---

## How It Works

**Before fix:**
1. User logs out → redirected to `/login`
2. User presses Back → browser shows cached `/dashboard` (no server request)
3. User continues using the app without being authenticated

**After fix:**
1. User logs out → redirected to `/login` (with no-cache headers)
2. User presses Back → browser sends fresh request to `/dashboard`
3. `auth` middleware checks session → no valid session → redirects to `/login`
4. User sees the login page

---

## Note

If the production site later switches to HTTPS, set `SESSION_SECURE_COOKIE=true` back for security. The no-cache headers should remain regardless.
