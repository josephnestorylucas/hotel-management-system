# Bug 001 — Booking History Missing on Guest Profile Page

**Date:** 2026-05-29
**Status:** Fixed

---

## Problem

The guest show page (`/guests/{id}`) only displayed **reservation history**. There was no **booking history** section at all. This meant:

- After a guest is **checked in** (booking created), you could NOT see the booking details on the guest profile
- After a guest is **checked out**, the booking was invisible on the guest page
- No way to tell if a guest has an active stay or past stays from the guest view

---

## Root Cause

The `GuestController@show` method only eager-loaded `reservations`:

```php
// BEFORE (broken)
$guest->load(['reservations.room.roomType', 'media']);
```

The `bookings` relationship was never loaded, and the view (`guests/show.blade.php`) had no booking history section.

---

## Fix

### 1. `app/Http/Controllers/GuestController.php` (line 100)

Added `bookings.room.roomType` to the eager load:

```php
// AFTER (fixed)
$guest->load(['reservations.room.roomType', 'bookings.room.roomType', 'media']);
```

### 2. `resources/views/guests/show.blade.php`

Added a full **Booking History** section after the Reservation History section. It shows:

- Booking number (clickable link to booking detail page)
- Room number + room type
- Check-in date
- Check-out date + number of nights
- Total amount
- Status badge (checked_in / checked_out / cancelled)
- Empty state message when no bookings exist

### 3. `resources/lang/en/guests.php`

Added English translation keys:
